# VAP-TEP Payload Model

## Purpose

This document defines the conceptual scientific payload expected within a Variant Annotation Pipeline Transitional Evidence Product (VAP-TEP).

The purpose of this document is not to define a transport specification.

The purpose is not to define a schema.

The purpose is not to define implementation details.

Instead, this document defines the scientific cargo that must be preserved and transported so that future systems can accurately reconstruct what VAP observed, what VAP inferred, and what contextual evidence existed at the time of execution.

This document should be interpreted as a conceptual payload model.

VAP-TEP payload semantics are owned by VAP.

The VAP-TEP sleeve determines how VAP output-contract artifacts are projected into transportable semantic cargo.

The TEP envelope is governed by the Truth Layer and must preserve identity, versioning, validation state, provenance, and source artifact manifest metadata.

# Payload Thesis

A VAP-TEP carries biological observations together with the contextual evidence required to preserve scientific interpretability.

The payload is not merely a collection of candidate variants.

The payload is not merely a summary report.

The payload is a preservation vehicle whose purpose is to ensure that future systems can reconstruct:

* what was observed
* how the observation was contextualized
* how the observation was interpreted
* how the observation was prioritized
* what uncertainty existed
* what future reinterpretation opportunities remain

Accordingly:

> A VAP-TEP transports observations plus the contextual layers required to keep those observations scientifically interpretable.

---

# Conceptual Payload Overview

A single VAP run package may generate one or more VAP-TEPs. VAP-TEP cardinality is an implementation decision, provided each TEP preserves source_package_id, source_identity_scope, source artifact manifest, provenance, and reconstructable evidence topology.

At a conceptual level, a VAP-TEP carries the following entity classes:

```text
Run Entity
    ↓

Sample Entity
    ↓

Observation Entity
    ↓

Variant Entity
    ↓

Genotype Entity
    ↓

Annotation Entity
    ↓

Population Context Entity
    ↓

Clinical Context Entity
    ↓                  ↘     

Coding Interpretation    Noncoding Interpretation
    Entity                Entity
    ↓                    ↙

Prioritization Entity
    ↓

Validation Entity
    ↓

Provenance Entity
    ↓

Uncertainty Entity
```

These entities collectively represent the scientific state generated by VAP.

No single entity is sufficient in isolation.

Scientific meaning emerges from the combination of these contextual layers.

A VAP-TEP must preserve a source artifact manifest linking transported evidence back to contributing VAP run artifacts, stages, fields, and semantic transformation lineage.

---

# Run Entity

## Purpose

Represents the execution context that produced the preserved evidence.

## Scientific Importance

Future systems must be able to determine:

* which execution produced the evidence
* which analytical context produced the evidence
* which repository state generated the evidence

The run entity establishes the temporal and computational context surrounding all downstream observations.

---

# Sample Entity

## Purpose

Represents the biological sample associated with preserved observations.

## Scientific Importance

Future systems must be able to determine:

* which sample was evaluated
* which observations belong to the sample
* which evidence surfaces originate from the sample

The sample entity provides the biological anchor for all downstream interpretation.

---

# Observation Entity

## Purpose

Represents the existence of an observed biological signal.

## Scientific Importance

The observation entity is the most fundamental preservation unit.

It answers:

```text
What was observed?
```

An observation may exist regardless of whether it receives strong interpretation support.

An observation may remain scientifically valuable even when current biological understanding is limited.

Observation preservation therefore precedes interpretation preservation.

---

# Variant Entity

## Purpose

Represents the biological variant associated with an observation.

## Scientific Importance

Future systems must be able to reconstruct the specific variant associated with preserved evidence.

The variant entity establishes the biological substrate upon which all downstream reasoning depends.

Without variant identity, future reinterpretation becomes impossible.

---

# Genotype Entity

## Purpose

Represents the observed genotype state associated with a variant.

## Scientific Importance

Future systems must be able to determine how the variant was observed within the sample.

Genotype state provides critical biological context for future disease modeling, inheritance reasoning, burden analyses, and reinterpretation frameworks.

Variant identity without genotype context is incomplete.

---

# Annotation Entity

## Purpose

Represents biological context assigned during annotation.

## Scientific Importance

The annotation entity captures what VAP knew about the observation at the time of execution.

Examples include:

* transcript context
* consequence context
* gene context
* annotation relationships
* mapping relationships

The annotation entity preserves historical interpretation context while allowing future reinterpretation.

---

# Population Context Entity

## Purpose

Represents population-level evidence associated with an observation.

## Scientific Importance

Population evidence contributes to rarity assessment, burden modeling, cohort comparison, and future disease interpretation.

Population context may evolve over time.

Preserving the population evidence available during execution enables historical reconstruction.

---

# Clinical Context Entity

## Purpose

Represents clinically relevant context available during execution.

## Scientific Importance

Future systems must understand the clinical interpretation landscape available when VAP executed.

Clinical context should be viewed as historical evidence rather than immutable truth.

Future reinterpretation may differ from current interpretation.

Preserving historical clinical context enables comparison across scientific eras.

---

# Coding Interpretation Entity

## Purpose

Represents coding-focused reasoning generated during VAP interpretation.

## Scientific Importance

The coding interpretation entity captures how VAP evaluated coding consequences associated with observed variants.

This entity preserves:

* coding classifications
* coding evidence assessments
* coding interpretation outcomes
* coding-related uncertainty

Coding interpretation remains important even when future systems apply different reasoning models.

---

# Noncoding Interpretation Entity

## Purpose

Represents noncoding-focused reasoning generated during VAP interpretation.

## Scientific Importance

The noncoding interpretation entity preserves contextual information associated with regulatory and noncoding evidence.

Current understanding of noncoding biology remains incomplete.

Accordingly, preservation of noncoding interpretation context is particularly important for future discovery.

This entity preserves:

* regulatory context
* noncoding evidence
* noncoding classifications
* proximity relationships
* interpretation uncertainty

Future systems may derive substantial meaning from evidence that appears weakly informative today.

---

# Prioritization Entity

## Purpose

Represents prioritization outcomes generated during VAP execution.

## Scientific Importance

Future systems must understand:

* which evidence was escalated
* which evidence was deprioritized
* why prioritization decisions occurred

The prioritization entity preserves historical decision context.

Importantly, prioritization status should not be confused with scientific importance.

Deprioritized evidence may remain highly relevant for future discovery.

---

# Validation Entity

## Purpose

Represents validation readiness assessments generated by VAP.

## Scientific Importance

The validation entity preserves the final review-oriented state generated by the workflow.

Future systems must understand:

* which evidence was considered validation-ready
* which evidence was not considered validation-ready
* why those decisions occurred
* reviewability context

Validation status provides important historical context but should not replace upstream observations.

---

# Provenance Entity

## Purpose

Represents the origin and lineage of preserved evidence.

## Scientific Importance

Future systems must be able to determine:

* where evidence originated
* how evidence was generated
* which execution produced the evidence
* how evidence moved through the workflow

Provenance is required for reproducibility, auditability, and scientific defensibility.

---

# Uncertainty Entity

## Purpose

Represents limitations associated with observations and interpretations.

## Scientific Importance

Future systems must distinguish between:

* known states
* unknown states
* ambiguous states
* unmapped states
* unsupported states
* unavailable states

Uncertainty is itself a scientific observation.

Preserving uncertainty protects future reinterpretation efforts from overconfidence and historical distortion.

A payload should be considered scientifically complete if future systems can determine:

```text
What was observed?
```

```text
Which sample contained the observation?
```

```text
What variant produced the observation?
```

```text
What genotype was observed?
```

```text
What annotation context existed?
```

```text
What interpretation context existed?
```

```text
What prioritization decisions occurred?
```

```text
What validation decisions occurred?
```

```text
What uncertainty existed?
```

```text

```

without requiring access to the original VAP execution environment.

# Edge Cases

Particular care should be taken to preserve:

* noncoding observations
* common variants
* weakly supported variants
* variants lacking annotation
* variants lacking clinical support
* deprioritized evidence
* ambiguous mappings
* conflicting interpretations
* healthy-control observations
* negative findings

These evidence classes may become disproportionately important for future discovery.

---

# Validation Strategy

A payload model should be considered successful if future systems can reconstruct:

1. the observed biological substrate
2. the contextual evidence available to VAP
3. the interpretation state generated by VAP
4. the prioritization state generated by VAP
5. the uncertainty state generated by VAP
6. the provenance lineage generated by VAP

without requiring re-execution of the workflow.

# Conclusion

A VAP-TEP is not merely a candidate transport mechanism.

It is a scientific preservation vehicle that carries observations together with the contextual evidence required to preserve interpretability.

The payload should preserve biological substrate, interpretation context, provenance, uncertainty, prioritization history, and validation state so that future systems can continue reasoning over VAP evidence long after the original execution environment no longer exists.
