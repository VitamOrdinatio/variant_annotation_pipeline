# VAP-TEP Implementation Plan

## Purpose

This document defines the implementation strategy for constructing the VAP Transitional Evidence Product (VAP-TEP).

The purpose of VAP-TEP construction is to preserve evidence lineage generated by VAP and transport that lineage to downstream ecosystem consumers.

The primary VAP-TEP v1 consumer is:

```text
VDB
```

Implementation must satisfy:

```text
docs/contracts/system/core/vap_tep_contract.md

docs/contracts/system/interfaces/vap_tep_transport_requirements.md

docs/contracts/system/validation/vap_tep_acceptance_criteria.md
```

---

# Goals

VAP-TEP v1 must:

* preserve Stage07 observation lineage
* preserve Stage08 normalization lineage
* preserve Stage08 routing lineage
* preserve Stage09 interpretation lineage
* preserve Stage10 interpretation lineage
* preserve Stage11 prioritization lineage
* preserve Stage12 validation lineage
* preserve Stage13 execution context
* generate machine-readable lineage metadata
* support deterministic downstream ingestion

---

# Non-Goals

VAP-TEP v1 will not:

* perform VDB ingestion
* perform RDGP ingestion
* perform evidence federation
* perform candidate discovery
* perform evidence ranking
* perform cross-run analysis

VAP-TEP is a preservation and transport layer.

Do not immediately serialize entities into a final JSON schema.

---

The implementation will proceed in four phases.

```text

    Validation Framework
```

## Deliverable

```text

```

Proposed v1 structure:

```text

├── entities/
│
├── observation/
├── normalization/
├── routing/
├── coding_interpretation/
├── noncoding_interpretation/
├── prioritization/
├── validation/

├── context/
│
```

```text

portable
deterministic
```

# V1 Exit Criteria

VAP-TEP v1 is considered complete when:

```text
A completed VAP run can produce:

    VAP-TEP package

that:

    passes all acceptance criteria

and

    can be consumed by VDB

without requiring access to the original VAP run directory.
```

The resulting artifact becomes the canonical preservation and transport substrate for VAP evidence within the broader repository ecosystem.
