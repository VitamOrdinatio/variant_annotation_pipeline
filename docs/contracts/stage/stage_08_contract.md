# Stage 08 — Filter & Partition Contract

**Pipeline:** Variant Annotation Pipeline (VAP)
**Stage:** 08
**Status:** Specification (Pre-Implementation)

---

# 🧭 Purpose

Stage 08 converts **raw VEP annotation output** into a **biologically structured, lossless, interpretation-ready dataset**.

It is the first stage that transitions VAP from:

```text
toolchain execution → biological interpretation system
```

---

# 🎯 Design Principles

1. **Lossless transformation**

   * No biologically plausible variants are permanently discarded
   * All original VEP data must remain recoverable

2. **Separation of concerns**

   * Stage 08 = classification + partitioning
   * Stage 09+ = interpretation + prioritization

3. **Transcript-aware design**

   * Do NOT collapse transcript-level consequences prematurely

4. **Deterministic output**

   * Same input → identical partition outputs

---

## Downstream Compatibility Requirements

Stage 08 outputs must preserve the minimum fields required for future VDB ingest and RDGP gene-level aggregation.

Required downstream-compatible fields:

- `sample_id`
- `run_id`
- `source_pipeline`
- `variant_id`
- `chromosome`
- `position`
- `reference_allele`
- `alternate_allele`
- `variant_type`
- `variant_class`
- `quality_flag`
- `gene_id`
- `gene_symbol`
- `transcript_id`
- `consequence`
- `impact_class`
- `clinical_significance`
- `clinvar_significance`
- `population_frequency`
- `gnomad_af`
- `exac_af`
- `thousand_genomes_af`
- `mito_flag`
- `epilepsy_flag`
- `annotation_source` (provenance field)
- `annotation_version` (provenance field)

Stage 08 must not rename these fields without documenting a schema mapping.

### Key rules to clarify fields

**1. consequence vs. impact_class** 

`consequence` holds fine-grained, categorical information about the variant pulled directly from vep, such as:
- missense_variant
- stop_gained
- splice_region_variant
- intron_variant

`impact_class` holds a coarse severity bin, such as:
- HIGH
- MODERATE
- LOW
- MODIFIER

Both must be preserved.


**2. Significance fields:**

`clinical_significance` must never overwrite or replace `clinvar_significance`.

  1. `clinvar_significance` = raw source truth (Pathogenic, Benign, etc.)
  2. `clinical_significance` = normalized interpretation field (aggregates from ClinVar and other sources)

Both must be preserved.


**3. population_frequency vs gnomad_af vs exac_af vs thousand_genomes_af fields:**

Aggregated:
- `population_frequency` holds a single summary value that is aggregated across source-specific AF fields

Source-specific AF fields (raw-evidence):
- `gnomad_af`
- `exac_af`
- `thousand_genomes_af`

All must be preserved.

`population_frequency` must be computed using a defined strategy:

Default `population_frequency` calculation for VAP v1:

```text
population_frequency = max(
  gnomad_af,
  exac_af,
  thousand_genomes_af
)
```

**Rationale:**
conservative estimate of population prevalence

---

## Phenotype Flags (Optional)

Fields such as:
- mito_flag
- epilepsy_flag

must be:

1. deterministically derived
2. backed by a defined source (e.g., GSC gene set, ClinVar annotation)
3. documented in a separate phenotype_flag_spec.md

Stage 08 must not infer phenotype flags heuristically.

## Field Naming Compatibility

Stage 08 may use verbose field names (e.g., chromosome, position),
but must provide a documented mapping to interface-required fields:

chrom ↔ chromosome  
pos ↔ position  
ref ↔ reference_allele  
alt ↔ alternate_allele

---

## Null Handling

Stage 08 must distinguish:

- `NA` = unavailable from annotation source
- empty string = parsing failure or malformed input
- `0` = observed numeric zero

Stage 08 must not convert missing allele frequency to rare.
Stage 08 must not convert missing clinical significance to benign.

# 🔄 Processing Logic

## Step 1 — Parse CSQ Field

* Expand VEP annotations into **one row per variant–transcript pair**
* Preserve ALL transcript consequences

Output:

```text
stage08_transcript_consequences.tsv
```

---

## Step 2 — Assign Variant Context

Each row must be labeled with:

```text
variant_context ∈ {
  coding,
  splice_region,
  regulatory,
  intronic,
  intergenic,
  unknown
}
```

---

## Example Classification Expectations

Stage 08 must correctly classify the following known Stage 07 examples:

### Example 1 — Noncoding transcript exon variant

Input:
- `variant_id`: `1:14522:G:A`
- `gene_symbol`: `WASH7P`
- `consequence`: `non_coding_transcript_exon_variant`
- `impact_class`: `MODIFIER`
- `variant_class`: `non-coding`

Expected:
- `variant_context`: `regulatory` or `noncoding_transcript`
- `variant_effect_severity`: `MODIFIER`
- `interpretability_status`: `needs_external_annotation`

### Example 2 — Missense variant

Input:
- `variant_id`: `1:953279:T:C`
- `gene_symbol`: `NOC2L`
- `consequence`: `missense_variant`
- `impact_class`: `MODERATE`
- `variant_class`: `coding`

Expected:
- `variant_context`: `coding`
- `variant_effect_severity`: `MODERATE`
- `interpretability_status`: `interpretable_now`

### Example 3 — Stop-gained variant

Input:
- `variant_id`: `1:48242556:G:T`
- `gene_symbol`: `SLC5A9`
- `consequence`: `stop_gained`
- `impact_class`: `HIGH`
- `variant_class`: `coding`

Expected:
- `variant_context`: `coding`
- `variant_effect_severity`: `HIGH`
- `interpretability_status`: `interpretable_now`

## Step 3 — Assign Effect Severity

Based on VEP `IMPACT` and `Consequence`:

```text
variant_effect_severity ∈ {
  HIGH,
  MODERATE,
  LOW,
  MODIFIER,
  UNKNOWN
}
```

Mapping must remain transparent and reproducible.

---

## Step 4 — Assign QC Status

```text
qc_status ∈ {
  pass,
  caution,
  fail
}
```

### Criteria

* Missing essential fields → fail
* Ambiguous annotation → caution
* Otherwise → pass

---

## Step 5 — Assign Derived Annotation Labels

Stage 08 may derive additional annotation labels from existing fields.
These labels must be deterministic, reproducible, and non-destructive.

### 5.1 Interpretability Status

interpretability_status ∈ {
  interpretable_now,
  needs_external_annotation,
  unsupported_currently
}

---

### 5.2 Frequency Status

Derived from:
- population_frequency
- gnomad_af
- exac_af
- thousand_genomes_af

frequency_status ∈ {
  common,
  low_frequency,
  rare,
  missing,
  unknown
}

#### Suggested thresholds (v1, configurable later):

- common: AF ≥ 0.05
- low_frequency: 0.01 ≤ AF < 0.05
- rare: AF < 0.01
- missing: no AF fields populated
- unknown: conflicting AF signals across sources

Stage 08 must preserve raw AF values alongside this label.

---

### 5.3 Clinical Status

Derived from:
- clinical_significance
- clinvar_significance

clinical_status ∈ {
  pathogenic,
  likely_pathogenic,
  vus,
  likely_benign,
  benign,
  conflicting,
  missing
}

#### Rules:

- conflicting: multiple annotation sources disagree
- missing: no clinical annotation available

Stage 08 must preserve raw clinical annotation fields.

Collapse transcript-level rows into variant-level summary:


* highest_impact

* splice_flag (boolean)

## Step 7 — Partition Outputs

Create the following files:

```text

splice_region_candidates.tsv
noncoding_candidates.tsv
qc_flagged.tsv
```

### Rules

* A variant may appear in multiple partitions if applicable
* No variant is dropped unless irreparably malformed

`stage08_vdb_ready_variants.tsv` is variant-centric and preserves one record per variant/transcript annotation suitable for VDB ingest.

`stage08_rdgp_gene_evidence_seed.tsv` is not final RDGP scoring. It is a minimal `(sample_id, gene_symbol/gene_id)` aggregation seed containing variant counts, highest impact, and contributing variant IDs.

# 🔒 Invariants

1. **Losslessness**

```text
Every input variant must be present in at least one output table
```

2. **Transcript fidelity**

```text
No transcript-level information is discarded
```

3. **Determinism**

```text
Repeated runs produce identical outputs
```

4. **Schema stability**

```text
Column names and formats must remain consistent
```

---

# ⚠️ Explicit Non-Goals

Stage 08 MUST NOT:

* Perform pathogenicity scoring
* Apply population frequency filtering
* Rank variants
* Integrate external databases (ClinVar, gnomAD, etc.)
* Collapse variants to a single “best” transcript
