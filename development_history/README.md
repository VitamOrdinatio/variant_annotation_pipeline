# Development History

## Purpose

This directory preserves important historical engineering artifacts generated during the iterative development of:

```text
variant_annotation_pipeline (VAP)
```

These materials document:

- exploratory execution harnesses
- checkpoint validation workflows
- environment setup diagnostics
- reproducibility investigations
- smoketesting
- telemetry-oriented debugging
- and evidence harvesting development

performed during the maturation of the VAP ecosystem.

---

# Why This Directory Exists

The VAP repository emphasizes:

```text
reproducibility
deterministic execution
provenance preservation
transparent engineering
and operational traceability
```

As a result, portions of the development process itself are intentionally preserved.

This directory acts as:

```text
engineering archaeology
```

rather than canonical production infrastructure.

---

# Important Distinction

The artifacts in this directory are:

- historically informative
- operationally meaningful
- useful for understanding repository evolution

but they are **NOT** considered:

- stable production interfaces
- canonical reusable tooling
- or guaranteed long-term APIs

Canonical operational code belongs elsewhere:

| Directory | Purpose |
|---|---|
| `src/` | reusable pipeline implementation |
| `scripts/` | supported operational tooling |
| `tests/` | formal validation/testing |
| `docs/` | architecture and documentation |

This directory instead preserves:

```text
historical development substrate
```

# Directory Structure

## exploratory_scripts/

Contains historical exploratory shell scripts used during:

- DEX 1.0 development
- checkpoint debugging
- environment validation
- installation troubleshooting
- schema probing
- biological evidence harvesting
- and staged smoketesting

These scripts are grouped by operational purpose.

### Subdirectories

| Directory | Purpose |
|---|---|
| `checkpoint_harnesses/` | staged execution and checkpoint probes |
| `environment_setup/` | environment validation and bootstrap checks |
| `evidence_harvest/` | reproducibility and biological overlay probes |
| `smoketests/` | rapid validation harnesses |
| `stage_probes/` | stage-specific schema/logic exploration |
| `tool_installation/` | VEP/GATK/tool installation workflows |

---
